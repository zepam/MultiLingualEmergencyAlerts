# JSON Schema Documentation: output_file.json

## Overview
The `output_file.json` file contains multilingual emergency alert translations generated by various AI clients and translation services. The data is organized hierarchically by client, language, alert type, prompt methodology, and individual alert entries.

## Structure

### Root Level
The root object contains keys for different AI clients and translation services:

```
{
  "chatgpt": {...},
  "gemini": {...},
  "deepseek": {...},
  "google_translate": {...},
  "deepL": {...}
}
```

**Clients:**
- `chatgpt` - OpenAI's ChatGPT model
- `gemini` - Google's Gemini model
- `deepseek` - Deepseek AI model
- `google_translate` - Google Translate service
- `deepL` - DeepL translation service

---

## Level 1: Client Object

Each client object maps to supported languages:

```
{
  "client_name": {
    "spanish": {...},
    "haitian_creole": {...},
    "vietnamese": {...},
    "arabic": {...},
    "chinese_traditional": {...}
  }
}
```

**Supported Languages:**
- `spanish`
- `haitian_creole`
- `vietnamese`
- `arabic`
- `chinese_traditional`

---

## Level 2: Language Object

Each language contains different emergency alert types:

```
{
  "language": {
    "flood": {...},
    "extreme_wind": {...},
    "fire": {...},
    "boil_water_notice": {...},
    "911_outage": {...}
  }
}
```

**Alert Types:**
- `flood` - Flood/inundation warnings
- `extreme_wind` - High wind warnings
- `fire` - Fire/wildfire warnings
- `boil_water_notice` - Water safety notices
- `911_outage` - Emergency service outage alerts

---

## Level 3: Alert Type Object

Each alert type contains multiple prompt methodologies used to generate translations:

```
{
  "alert_type": {
    "prompt_simple_360.txt": [...],
    "prompt_persona_nolimit.txt": [...],
    "prompt_one_shot_nolimit.txt": [...],
    "prompt_cross_lingual_alignment_nolimit.txt": [...],
    "prompt_chain_of_translation_360.txt": [...],
    "prompt_one_shot_360.txt": [...],
    "prompt_simple_nolimit.txt": [...],
    "prompt_cross_lingual_alignment_360.txt": [...],
    "prompt_chain_of_translation_nolimit.txt": [...],
    "prompt_persona_360.txt": [...]
  }
}
```

**Prompt Methodologies:**

| Prompt | Description |
|--------|-------------|
| `prompt_simple_360.txt` | Direct, simple translation prompt (360 character limit) |
| `prompt_simple_nolimit.txt` | Direct, simple translation prompt (no character limit) |
| `prompt_persona_360.txt` | Translation using a persona/role-based approach (360 limit) |
| `prompt_persona_nolimit.txt` | Translation using a persona/role-based approach (no limit) |
| `prompt_one_shot_360.txt` | One-shot learning approach (360 limit) |
| `prompt_one_shot_nolimit.txt` | One-shot learning approach (no limit) |
| `prompt_chain_of_translation_360.txt` | Chain-of-thought translation approach (360 limit) |
| `prompt_chain_of_translation_nolimit.txt` | Chain-of-thought translation approach (no limit) |
| `prompt_cross_lingual_alignment_360.txt` | Cross-lingual alignment methodology (360 limit) |
| `prompt_cross_lingual_alignment_nolimit.txt` | Cross-lingual alignment methodology (no limit) |

**Prompt Naming Convention:**
- Suffix `_360` indicates a 360-character limit constraint
- Suffix `_nolimit` indicates no character limit constraint

---

## Level 4: Alert Entry Array

Each prompt key contains an array of alert entries:

```
[
  {
    "text": "Translation text here",
    "date": "YYYY-MM-DD"
  },
  ...
]
```

### Entry Object

| Field | Type | Description |
|-------|------|-------------|
| `text` | String | The translated emergency alert message. May contain Unicode emoji and special characters. |
| `date` | String | ISO 8601 date format (YYYY-MM-DD) indicating when the translation was generated. |

**Example Entry:**
```json
{
  "text": "🚨 ALERTA DE INUNDACIÓN 🚨 Lluvias intensas provocan desbordes en zonas bajas. Evite áreas inundadas, busque refugio en lugares altos y siga indicaciones de autoridades. No conduzca por calles anegadas. Mantenga radio o teléfono encendidos para actualizaciones.",
  "date": "2025-09-13"
}
```

---

## Complete Hierarchical Path

A complete path to an entry would be:
```
root → client → language → alert_type → prompt_methodology → entry_index
```

**Example:**
```
output_file.json
└─ chatgpt
   └─ spanish
      └─ flood
         └─ prompt_simple_360.txt
            └─ [0]
               ├─ text: "🚨 ALERTA DE INUNDACIÓN..."
               └─ date: "2025-09-13"
```

---

## Key Characteristics

1. **Multi-level Hierarchy**: Data is organized across 4 hierarchical levels (client → language → alert_type → prompt_methodology)

2. **Multiple Clients**: Compares outputs from 5 different AI/translation services to evaluate quality and consistency

3. **Multiple Languages**: Supports 5 different target languages for emergency alerts

4. **Multiple Alert Types**: Covers 5 different types of emergency scenarios

5. **Multiple Prompt Strategies**: Tests 10 different prompting methodologies to evaluate which produces the best translations

6. **Temporal Data**: Each entry includes a generation date, allowing analysis of consistency over time

7. **Varied Output Lengths**: Some prompts have character limits (360) while others have no constraints

---

## Use Cases

- **Quality Evaluation**: Compare translation quality across clients and prompting methods
- **Consistency Analysis**: Track how different approaches generate consistent emergency messaging
- **Language Coverage**: Verify coverage across multiple languages
- **Temporal Trends**: Analyze how outputs change over time periods
- **Model Comparison**: Directly compare ChatGPT, Gemini, Deepseek, Google Translate, and DeepL
- **Prompt Engineering**: Evaluate which prompting methodology produces the best results

## JSON Schema (JSON Schema Format)

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "Multilingual Emergency Alerts",
  "type": "object",
  "properties": {
    "client": {
      "type": "object",
      "properties": {
        "language": {
          "type": "object",
          "properties": {
            "alert_type": {
              "type": "object",
              "properties": {
                "prompt_methodology": {
                  "type": "array",
                  "items": {
                    "type": "object",
                    "properties": {
                      "text": {
                        "type": "string",
                        "description": "Translated emergency alert message"
                      },
                      "date": {
                        "type": "string",
                        "pattern": "^\\d{4}-\\d{2}-\\d{2}$",
                        "description": "ISO 8601 date (YYYY-MM-DD)"
                      }
                    },
                    "required": ["text", "date"]
                  }
                }
              }
            }
          }
        }
      }
    }
  },
  "additionalProperties": true
}
```

## Notes

- The file structure supports flexible querying by client, language, alert type, or prompting method
- Dates range from 2025-2026, indicating future-dated test scenarios
- Text entries may contain emoji, special characters, and formatting specific to the target language
- All entries appear to be original translations rather than cached or copied values
- The schema is extensible—new clients, languages, alert types, or prompts can be added without breaking the structure
